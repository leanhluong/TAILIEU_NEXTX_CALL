**9:00 - 10:00: User Story Template**

# [US-XXX] Title

**Epic:** [Epic Name]
**Priority:** Must Have / Should Have / Could Have / Won't Have
**Story Points:** X

## User Story
As a: [Role]
I want: [Goal]
So that: [Benefit]

## Acceptance Criteria
✅ [Criterion 1]
✅ [Criterion 2]
✅ [Criterion 3]

- [Rule 1]
- [Rule 2]

## Test Scenarios

2. [Scenario 2]

[Additional notes]

## 💡 TIPS & BEST PRACTICES

### User Story Writing
✅ **DO:**
- Use INVEST criteria (Independent, Negotiable, Valuable, Estimable, Small, Testable)
- Write from user perspective
- Include clear Acceptance Criteria
- Add test scenarios
- Link dependencies

❌ **DON'T:**
- Write technical implementation
- Make stories too large
- Forget edge cases
- Skip business rules
- Ignore stakeholder input

### UAT Testing
✅ **DO:**
- Test happy path first
- Test error scenarios
- Test edge cases
- Document steps clearly
- Report bugs immediately

❌ **DON'T:**
- Skip test cases
- Assume it works
- Test only once
- Ignore minor bugs
- Delay reporting
